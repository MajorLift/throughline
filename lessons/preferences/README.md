## Format

```markdown
# {Preference Name}

**Preference:** {what you prefer}

**Why:** {rationale}

**Override:** {when to ignore this}
```

## Examples

- "Lead with the answer, skip preamble"
- "Use tables for comparing 3+ items"
- "Prefer concrete examples over abstract explanations"
- "When explaining, show code first, then explain"
- "Keep responses concise unless I ask for depth"

## Adding Preferences

Create a new file: `{preference-name}.md`

Or use: `/learn I prefer {X}` or `/learn preference: {description}`
